Problem Statements

Choose one out of 11 total Problem Statements during registration. Allocation is first-come, first-served.

PS-01 {{ ps_counts.get('PS-01', 0) }}/10 Teams

Smart Traffic Management System

Urban areas suffer from high congestion because traffic signals operate on static timers regardless of actual vehicle density. Participants must develop an AI-driven Adaptive Signal Control System that analyzes live traffic feeds to dynamically adjust "Green Light" durations. The system should prioritize emergency vehicles (ambulances/fire trucks) and reduce "dead-time" at empty intersections.

Tech Stack
  • Computer Vision: YOLO (v8/v10), OpenCV
  • Programming: Python
  • Simulation: SUMO or Pygame
  • Deployment: Flask/FastAPI
Expected Output
  • Real-time vehicle count dashboard
  • Waiting time reduction simulation
  • Emergency "Green Wave" triggering
PS-02 {{ ps_counts.get('PS-02', 0) }}/10 Teams

Railway Ticket Management & Predictive Analysis

Traditional railway systems struggle with "Waitlist" uncertainty and seat wastage. Participants need to create an Intelligent Booking & Prediction Engine. This system should not only manage bookings and cancellations but also use historical data to provide a PNR Confirmation Probability. Additionally, it should suggest "Alternative Route/Train" options if the primary choice is unavailable.

Tech Stack
  • Backend: Java (Spring Boot) or Node.js
  • Database: PostgreSQL or MongoDB
  • Data Science: Scikit-learn, Pandas, XGBoost
  • Frontend: React.js or Angular
Expected Output
  • Functional booking portal with PNR
  • Waitlist prediction model (>80% accuracy)
  • Dynamic Seat Allocation algorithm
PS-03 {{ ps_counts.get('PS-03', 0) }}/10 Teams

AI Content Detector

With the rise of Large Language Models (LLMs), distinguishing between human-written and AI- generated content is critical for academic integrity and misinformation control. Participants must build a Deep Learning Classifier that detects text generated by models like GPT-4, Claude, or Gemini. The tool should provide a "Perplexity" score and highlight specific segments likely to be AI- generated.

Tech Stack
  • NLP: Hugging Face, PyTorch/TensorFlow
  • Models: RoBERTa, BERT, or T5
  • Data: Web-scraped / Deepfake Text dataset
  • Frontend: Streamlit or Tailwind CSS
Expected Output
  • Web interface for text analysis
  • AI vs Human Probability Score
  • Model performance report (F1, Precision)
PS-04 {{ ps_counts.get('PS-04', 0) }}/10 Teams

AI-Powered Smart Agriculture (IoT & CV)

Farmers often face crop loss due to undetected pests or nutrient deficiencies. Participants must build a Crop Health Monitoring System that analyzes images of leaves to identify diseases and recommend specific fertilizers or pesticides. The system should also predict the "Optimal Harvest Date" based on historical weather patterns.

Tech Stack
  • DL Frameworks: TensorFlow/Keras
  • Datasets: PlantVillage dataset
  • Mobile/Web: Flutter or React Native
Expected Output
  • Mobile app for leaf diagnosis
  • Soil moisture & weather dashboard
PS-05 {{ ps_counts.get('PS-05', 0) }}/10 Teams

Blockchain-Based Supply Chain for Pharmacy

The pharmaceutical industry is plagued by counterfeit drugs. Participants are tasked with creating a Transparent Drug Tracking System using Blockchain. Every step from the manufacturer to the local pharmacy must be logged on a ledger to ensure the medicine's authenticity.

Tech Stack
  • Blockchain: Ethereum, Solidity
  • Web3: Ethers.js or Web3.js
  • Backend: Node.js
Expected Output
  • dApp for "Chain of Custody" tracking
  • Immutable drug storage logs
PS-06 {{ ps_counts.get('PS-06', 0) }}/10 Teams

Personalized Mental Health Chatbot (NLP)

With rising stress levels, many students don't have immediate access to counseling. Build an Empathetic AI Companion that uses Sentiment Analysis to gauge a user's mood. It should provide coping mechanisms for anxiety/stress and, most importantly, trigger an "Emergency Alert" to a human counselor if it detects signs of self-harm.

Tech Stack
  • NLP: OpenAI API (GPT-4) or Llama 3
  • Sentiment: VADER or TextBlob
  • Frontend: Next.js
Expected Output
  • Real-time responsive chat interface
  • Weekly emotional trend tracker
PS-07 {{ ps_counts.get('PS-07', 0) }}/10 Teams

Fraud Detection in UPI/Digital Payments

Digital payment fraud is evolving rapidly. Participants must develop a Real-Time Fraud Detection Engine that flags suspicious transactions based on anomalies (e.g., unusual location, sudden high-value transfer, or rapid-fire small transactions).

Tech Stack
  • ML: Isolation Forest, Random Forest
  • Stream: Apache Kafka or Spark
  • Database: Redis (fast lookups)
Expected Output
  • Transaction classification system
  • False Positive rate report
PS-08 {{ ps_counts.get('PS-08', 0) }}/10 Teams

Automated Resume Screener & Job Matcher

HR departments receive thousands of resumes that don't match job descriptions (JD). Build a Resume Parser that extracts skills, experience, and education, then ranks candidates based on a "Match Score" against a specific JD using Vector Embeddings.

Tech Stack
  • Parsing: PyResparser or Spacy
  • Search: Cosine Similarity, FAISS
  • Frontend: React.js
Expected Output
  • Recruiter dashboard for PDF sorting
  • Candidate "Skill Gap" analysis
PS-09 {{ ps_counts.get('PS-09', 0) }}/10 Teams

Decentralized Energy Grid Simulator

In a smart city, households with solar panels should be able to sell excess electricity to neighbors. Create a P2P Energy Trading Platform that uses a bidding algorithm to balance the local grid's supply and demand without a central authority.

Tech Stack
  • Simulation: Python (Mesa)
  • Logic: Nash Equilibrium algorithms
  • UI: D3.js grid visualization
Expected Output
  • Visual energy flow map
  • Dynamic pricing model

Predictive Healthcare Analytics

Build a model to predict patient readmission rates using historical medical records and demographic data.

Tech Stack
  • Programming: Python (Pandas, NumPy)
  • ML Libraries: Scikit-learn, XGBoost
  • Visualization: Matplotlib, Seaborn
  • Deployment: Streamlit or Flask
Expected Output
  • Readmission prediction model
  • Feature importance analysis
  • Healthcare professional dashboard

Air Quality Index Prediction

Build a forecasting model for AQI levels in urban areas using meteorological and pollution data.

Tech Stack
  • Programming: Python
  • Time-series: ARIMA, Prophet, or LSTM
  • Data: OpenAQ API or similar
  • Visualization: Plotly or D3.js
Expected Output
  • Real-time AQI forecasting model
  • Spatiotemporal heatmaps
  • Pollution alert system

Hackathon Information

About the Event

Datathon 2026, organized by GirlScript MITAOE, is a Software Hackathon for undergraduate students. Participants will solve real-world challenges across various domains for a prize pool and internship opportunities.

Eligibility & Format

  • Open to all undergraduate students.
  • Team Size: 1-4 members (inter-college & inter-specialization teams allowed).
  • Industry-related Problem Statements will be provided.

Event Process

  • Problem statement selection.
  • Submission of solution video & PPT for shortlisting.
  • Grand Finale with live demos & presentations.

Rules & Judging

  • Teams can propose their own problem statement.
  • Strict deadlines; no late submissions allowed.
  • Plagiarism will lead to immediate disqualification.
  • Solutions are judged on: innovation, feasibility, scalability, technical execution, and presentation clarity.
